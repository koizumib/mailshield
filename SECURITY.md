## 脆弱性の報告

セキュリティ上の脆弱性を発見した場合は **公開 Issue ではなく** 以下の方法で報告してください。

### 報告方法

**GitHub Private Vulnerability Reporting** を使用してください:

1. [Security タブ](https://github.com/koizumib/mailshield/security) を開く
2. "Report a vulnerability" をクリック
3. 詳細を記入して送信

GitHub Private Vulnerability Reporting が使えない場合は、メンテナーのメールアドレス（bakuk0801@gmail.com）に直接連絡してください。

### 報告に含める情報

- 脆弱性の種類（例: RCE・SSRF・SQLインジェクション・認証バイパス）
- 影響を受けるコンポーネントとバージョン
- 再現手順（概念実証コードがあれば添付）
- 影響範囲の評価

---

## 対応の流れ

| タイミング | 内容 |
|-----------|------|
| 報告受領後 3 営業日以内 | 受領確認の返信 |
| 報告受領後 14 日以内 | 初期評価と対応方針の通知 |
| 対応方針合意後 | 修正の実装・テスト・リリース |
| リリース後 | CVE 番号の取得（必要に応じて）と公開開示 |

修正が完了するまで脆弱性の詳細を公開しないようご協力をお願いします（Coordinated Disclosure）。

---

## スコープ

以下はセキュリティ報告の対象です:

- smtp-gateway / api-server の認証・認可バイパス
- EML 処理・Lua ワーカー実行における任意コード実行
- テナント分離の破壊（他テナントのデータへのアクセス）
- マルチテナント環境での情報漏洩
- SQLインジェクション・XSS・SSRF

以下は対象外です:

- 同梱の開発用ツール（Postfix・Mailpit 等）自体の脆弱性
- ユーザーが意図的に設定したポリシーによる動作
- DoS（このプロジェクトはレート制限の実装を保証しません）
- 最新でないバージョンへの報告
